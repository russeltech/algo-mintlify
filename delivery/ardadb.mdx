## Running Your First Query

The example below aggregates daily trading volume across all tickers over a six-week window and filters for high-activity names. It illustrates ArdaDB's performance on a typical research-scale scan:

```sql
SELECT
    TradeDate,
    Ticker,
    sum(Volume) AS DailyVolume
FROM USEquityMarketData.TradeAndQuoteMinuteBar
WHERE (TradeDate >= '2023-01-18') AND (TradeDate <= '2023-02-28')
GROUP BY
    TradeDate, Ticker
HAVING DailyVolume > 10000000
ORDER BY TradeDate, DailyVolume DESC
```

This query returns **2,051 rows** in **6.15 seconds**, processing **273.55 million rows** at **44.48 million rows/sec** — on a single ArdaDB instance with no pre-aggregation.

## Storage Architecture

ArdaDB uses a tiered storage model designed for both performance and cost efficiency. Each instance mounts private EBS volumes for hot query data alongside an S3 backing tier for the full historical archive. Enterprise clients can share a dedicated multi-team S3 storage tier, allowing multiple users within an organization to query the same underlying dataset without duplicating storage costs.

## Connecting to ArdaDB

ArdaDB exposes a standard ClickHouse HTTP interface and native TCP interface, so any ClickHouse-compatible client works out of the box. Popular options include DBeaver, Tableau, the Python `clickhouse-driver` library, and Pandas via `clickhouse-connect`.

The Python example below connects to ArdaDB and loads a day of MSFT minute bars directly into a DataFrame:

```python
import clickhouse_connect

client = clickhouse_connect.get_client(
    host='your-ardadb-host.algoseek.com',
    port=8443,
    username='your_username',
    password='your_password',
    secure=True
)

result = client.query(
    """
    SELECT TradeDate, Ticker, OpenPrice, ClosePrice, Volume
    FROM USEquityMarketData.TradeAndQuoteMinuteBar
    WHERE Ticker = 'MSFT'
      AND TradeDate = '2024-06-01'
    ORDER BY BarTime
    """
)
df = result.result_df()
print(df.head())
```

Replace `your-ardadb-host.algoseek.com`, `your_username`, and `your_password` with the credentials provided in the AlgoSeek Console after your ArdaDB subscription is activated.
